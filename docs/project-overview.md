# Gonka Project Overview

## Введение

**Gonka** - это децентрализованная AI-инфраструктурная платформа, предоставляющая открытую альтернативу централизованным облачным провайдерам для обучения и инференса AI-моделей. Платформа направлена на демократизацию доступа к вычислительной мощности для AI-задач, обеспечивая экономическую эффективность и справедливую компенсацию провайдерам оборудования.

**Миссия:** Позволить разработчикам запускать инференс и обучение LLM на децентрализованной сети с вознаграждением провайдеров оборудования (нод) на основе их вычислительного вклада.

## Технологический стек

### Языки программирования

- **Go (1.24.2)** - блокчейн-нода и децентрализованный API-сервер
- **Python (^3.10)** - ML-нода для выполнения AI-задач (PyTorch, vLLM)
- **Kotlin** - интеграционное тестирование (Testermint)
- **Protocol Buffers** - определения сообщений и сериализация (73 proto-файла)

### Ключевые технологии

#### Блокчейн и консенсус
- **Cosmos SDK v0.53.3** - фреймворк для смарт-контрактов и управления состоянием
- **CometBFT v0.38.17** - Byzantine-fault tolerant консенсус (форк Tendermint)
- **Protocol Buffers** - сериализация блокчейн-данных
- **IBC (Inter-Blockchain Communication)** - поддержка кросс-чейн взаимодействия

#### API и сервисы
- **gRPC** - высокопроизводительный RPC для внутренней коммуникации
- **Echo v4** - HTTP web framework (публичный REST API)
- **NATS v2** - брокер сообщений для межсервисной коммуникации
- **PostgreSQL** - персистентное хранилище состояния (опционально)
- **SQLite** - файловое хранилище (fallback)

#### Криптография
- **BLS Signatures** - пороговые подписи для DKG
- **ECDSA (secp256k1)** - подписание транзакций
- **gnark-crypto** - криптографические примитивы

#### AI/ML компоненты
- **PyTorch** - фреймворк для глубокого обучения
- **vLLM** - высокопроизводительный инференс LLM
- **FastAPI** - Python async web framework для ML API
- **DiLoCo** - периодическая синхронизация для geo-распределенного обучения

## Архитектура

Gonka использует модульную контейнеризированную микросервисную архитектуру с тремя основными уровнями:

```
┌─────────────────────────────────────────────┐
│      Chain Node (Blockchain Layer)          │
│  - Consensus & State Management             │
│  - Proof of Work 2.0 Implementation         │
│  - Built on: Cosmos SDK + Tendermint        │
└─────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────┐
│    API Node (Coordination Layer)             │
│  - REST/gRPC Endpoints                      │
│  - Task Orchestration & Routing             │
│  - Validation & Verification                │
│  - Built in Go, Communication via NATS      │
└─────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────┐
│      ML Node (Compute Layer)                │
│  - AI Model Training & Inference            │
│  - Proof of Compute Execution               │
│  - GPU/CUDA Support                         │
│  - Built in Python (FastAPI)                │
└─────────────────────────────────────────────┘
```

## Основные компоненты

### A. Inference Chain (`/inference-chain`)

Блокчейн-нода, построенная на Cosmos SDK и CometBFT консенсусе. Содержит 8 кастомных модулей:

| Модуль | Назначение |
|--------|-----------|
| **BLS** | Схема пороговых подписей для распределенного консенсуса |
| **Inference** | Управление задачами инференса и валидация |
| **Bookkeeper** | Финансовый учет и распределение вознаграждений |
| **Collateral** | Управление стейкингом и залогом |
| **Restrictions** | Сетевые ограничения и rate limiting |
| **GenesisTransfer** | Начальное распределение токенов |
| **StreamVesting** | Временная блокировка токенов (vesting) |

**Ключевые обязанности:**
- Механизм консенсуса (Proof of Work 2.0)
- Управление состоянием задач, участников и эпох
- Расчет и распределение вознаграждений
- Переходы между эпохами и фазами

### B. Decentralized API (`/decentralized-api`)

Уровень оркестрации и координации. Основные компоненты:

| Компонент | Ответственность |
|-----------|-----------------|
| **Broker** | Маршрутизация задач между chain, API и ML нодами |
| **Chainphase** | Отслеживание фаз эпох блокчейна |
| **PoC Orchestrator** | Управление выполнением Proof of Compute |
| **Event Listener** | Обработка событий блокчейна |
| **Validation** | Реализация логики верификации задач |
| **Training** | Координация распределенного обучения моделей |
| **ModelManager** | Мониторинг здоровья ML нод |
| **Servers** | Public REST, Admin, ML Node, Training gRPC |

**Типы серверов:**
- **Public Server** - REST API для клиентов
- **ML Server** - коммуникация с ML нодами
- **Admin Server** - административные операции
- **Training Server** - gRPC для обучения моделей

### C. MLNode (`/mlnode`)

Python-based окружение для выполнения вычислений с модульными пакетами:

| Пакет | Назначение |
|-------|-----------|
| **common** | Общие утилиты и структуры данных |
| **pow** | Логика вычисления Proof-of-Work |
| **train** | Реализация обучения моделей (на базе PyTorch) |
| **api** | FastAPI REST сервер, экспонирующий все сервисы |

**Возможности:**
- Docker контейнеризация с поддержкой NVIDIA CUDA
- vLLM для оптимизации инференса
- PyTorch для обучения моделей
- Poetry для управления зависимостями

### D. Testermint (`/testermint`)

Kotlin-based фреймворк интеграционного тестирования:
- Симуляция полных сетевых окружений с Docker
- Использование WireMock для мокирования внешних сервисов
- Тестирование полного жизненного цикла задач: inference → validation → rewards
- Поддержка параллельного выполнения тестов

**Ключевые концепции:**
- `LocalInferencePair` - симуляция отдельных участников сети
- Тестирование multi-node кластеров с детерминированными сценариями
- Комплексное логирование с интеграцией `lnav`

### E. Вспомогательные компоненты

| Компонент | Назначение |
|-----------|-----------|
| **Proxy** | Nginx reverse proxy для маршрутизации API |
| **Proxy-SSL** | SSL терминация и управление сертификатами |
| **Bridge** | Реализация кросс-чейн моста токенов |
| **Client-libs** | Python клиентская библиотека для взаимодействия с сетью |
| **TMKMS** | Tendermint Key Management Service для подписи валидатора |

## Основная функциональность

### 1. Децентрализованный инференс и обучение

- Разработчики отправляют запросы на инференс и обучение
- Задачи валидируются и маршрутизируются на соответствующие ML ноды
- Результаты верифицируются несколькими пирами
- Вознаграждения распределяются на основе вычислительного вклада

### 2. Proof of Work 2.0

Инновационный механизм консенсуса, использующий вычисления на transformer-моделях:
- Временные "гонки" (Races), где участники конкурируют в решении AI-задач
- Вес голоса определяется успешными вычислениями
- Прямая корреляция между вычислительной мощностью и влиянием в управлении
- Обеспечивает ~100% утилизацию ресурсов для продуктивных AI-задач

### 3. Случайная верификация задач

- Вероятностная верификация вместо избыточной проверки
- Криптографически безопасный случайный выбор задач
- Проверяется только 1-10% задач, снижая накладные расходы
- Ноды с большим весом голоса несут большую ответственность за валидацию

### 4. Распределенное обучение моделей (DiLoCo)

- Geo-распределенное обучение через независимых провайдеров оборудования
- Периодическая синхронизация для минимизации коммуникационных накладных расходов
- Отказоустойчивая координация без централизованного надзора
- Статистическая валидация с учетом недетерминизма обучения

### 5. Система репутации

- Новые ноды начинают с нулевой репутацией
- Репутация растет при честном поведении
- Частая верификация для нод с низкой репутацией
- Штрафы репутации за обман/неверные результаты
- Стимулирует долгосрочную честность

### 6. Экономический слой

- Требование залога для участников
- Распределение вознаграждений на основе предоставленных вычислений
- Интеграция с Bitcoin для экономической безопасности
- Stream vesting для графиков разблокировки токенов
- Расчеты по счетам и финансовая сверка

## Workflow

```
Разработчик отправляет задачу
        ↓
Chain валидирует запрос и выбирает участника
        ↓
API маршрутизирует задачу на ML ноду
        ↓
ML нода выполняет inference/training
        ↓
API собирает результат
        ↓
Выбранные пиры верифицируют результат (случайно)
        ↓
Консенсус большинства → Распределение вознаграждений
        ↓
Обновление репутации и корректировка залога
```

## Разработка и развертывание

### Процесс сборки

- Оркестрация через Makefile для multi-компонентной сборки
- Docker контейнеризация для всех компонентов
- Раздельная сборка для chain, API и ML нод
- Статическая сборка бинарников с musl libc для портативности

### Стратегия тестирования

- Unit-тесты для отдельных компонентов
- GPU-enabled unit тесты для ML операций
- Интеграционные тесты через Testermint фреймворк
- End-to-end тестирование с полной симуляцией Docker кластера

### Модель развертывания

- Микросервисная архитектура через Docker контейнеры
- Каждый компонент развертывается независимо
- Горизонтальное масштабирование для ML нод
- Cosmovisor для обновлений chain без простоев

## Заключение

Gonka - это сложная, production-ready распределенная AI-вычислительная сеть, объединяющая блокчейн консенсус, распределенные вычисления и криптографическую верификацию в единую систему, предназначенную для демократизации доступа к AI-инфраструктуре.
